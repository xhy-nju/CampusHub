# CampusHub 需求说明

## 1. 项目定位

CampusHub 是一个校园互助服务平台，用于集中承接校内学生的日常互助需求，例如快递代取、学习辅导、二手物品交换、活动组队等。

平台目标：

- 让需求方能方便发布需求
- 让服务方能快速发现并接取需求
- 通过订单、评价和信用机制建立信任
- 为管理员提供基础治理能力

## 2. 用户角色

### 普通用户

同一个普通用户可以同时扮演需求方和服务方。

需求方能力：

- 发布需求
- 管理自己发布的需求
- 确认服务方
- 确认订单完成
- 评价服务方

服务方能力：

- 浏览需求
- 申请接单
- 管理自己接取的订单
- 标记服务完成
- 评价需求方

### 管理员

管理员负责平台治理。

- 查看用户
- 禁用用户
- 审核需求内容
- 查看订单和分类统计

## 3. 核心模块

### 用户管理

- 注册
- 登录
- 当前用户信息
- 个人资料管理
- 用户状态管理

### 需求发布与浏览

- 发布需求
- 需求列表
- 分类筛选
- 关键词搜索
- 需求详情
- 需求取消

需求字段：

- 标题
- 描述
- 分类
- 地点
- 期望时间
- 报酬或交换条件

### 订单流程管理

核心流程：

```text
需求发布 -> 服务方申请接单 -> 需求方确认 -> 进行中 -> 服务方完成 -> 需求方确认完成 -> 双方评价
```

必须保证：

- 用户不能接自己的需求
- 非订单双方不能操作订单
- 状态只能按合法顺序流转
- 已完成订单不能再次修改状态

### 评价与信用

- 订单完成后双方互评
- 评分范围为 1-5 星
- 支持文字评价
- 基于订单行为生成信用分记录

### 消息通知

站内通知包括：

- 接单申请通知
- 接单确认通知
- 订单状态变化通知
- 评价通知

即时通讯作为后续增强功能。

### 管理后台

- 用户管理
- 需求审核
- 基础统计

## 4. 非功能需求

- 性能：核心页面加载时间目标小于 2 秒
- 安全：密码加密存储，防止 SQL 注入和 XSS
- 可用性：核心操作路径不超过 3 步
- 可维护性：核心模块应有清晰分层和必要测试

## 5. MVP 范围

MVP 必须包含：

- 用户注册登录
- 需求发布、列表、详情、筛选
- 接单和订单完成流程

MVP 可以暂缓：

- 智能推荐
- 即时通讯
- 完整数据看板
- 自动部署 CD
